title: "Merchant BVN KYC"
description: "Basic and enhanced BVN verification for merchants"
---

Use these endpoints to **verify a customer’s Bank Verification Number (BVN)** as part of your merchant KYC workflow. Both basic and enhanced modes are supported.

Base paths:

- **Sandbox**: `https://sandbox.payvessel.com/kyc`
- **Production**: `https://api.payvessel.com/kyc`

All paths below are relative to the base URL.

## Basic BVN verification

<Badge variant="success">**POST**</Badge> `/api/v1/merchant/bvn/basic`

Send a BVN and minimal identity fields to confirm that the BVN is valid and belongs to the customer.

Typical request body fields:

- `bvn` – 11‑digit BVN  
- Optional identity fields such as name, date of birth, and phone number for extra matching  
- `reference` – unique reference for this verification

Basic BVN is suitable when you only need to confirm that the BVN is valid and linked to the customer.

## Enhanced BVN verification

<Badge variant="success">**POST**</Badge> `/api/v1/merchant/bvn/enhanced`

Enhanced verification returns a **richer profile** for stricter compliance checks.

Typical request body fields:

- `bvn` – 11‑digit BVN  
- Additional KYC attributes as required by your risk policy  
- `reference` – unique verification reference

Enhanced BVN is recommended for onboarding that requires full KYC (e.g. higher limits, business accounts).

## Example (basic BVN)

<CodeGroup>
```bash cURL
curl -X POST "https://sandbox.payvessel.com/kyc/api/v1/merchant/bvn/basic" \
  -H "Content-Type: application/json" \
  -d '{
    "bvn": "12345678901",
    "reference": "BVN_BASIC_2026_001"
  }'
```

```json Response
{
  "status": true,
  "message": "BVN verified",
  "data": {
    "bvn": "12345678901",
    "first_name": "JOHN",
    "last_name": "DOE",
    "date_of_birth": "1990-05-15",
    "phone_number": "+2348012345678"
  }
}
```
</CodeGroup>

## Example (enhanced BVN)

<CodeGroup>
```bash cURL
curl -X POST "https://sandbox.payvessel.com/kyc/api/v1/merchant/bvn/enhanced" \
  -H "Content-Type: application/json" \
  -d '{
    "bvn": "12345678901",
    "reference": "BVN_ENH_2026_001",
    "metadata": {
      "customer_id": "CUST_12345"
    }
  }'
```

```json Response
{
  "status": true,
  "message": "Enhanced BVN verified",
  "data": {
    "bvn": "12345678901",
    "first_name": "JOHN",
    "last_name": "DOE",
    "date_of_birth": "1990-05-15",
    "phone_number": "+2348012345678",
    "address": "123 MAIN STREET, LAGOS",
    "additional_info": {}
  }
}
```
</CodeGroup>
