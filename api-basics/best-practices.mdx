## Security Best Practices

### 🔐 API Key Management

<AccordionGroup>
  <Accordion icon="key" title="Secure Storage">
    **Do:**
    - Store API keys in environment variables
    - Use secure key management services (AWS KMS, Azure Key Vault)
    - Rotate keys regularly (every 90 days)
    - Use different keys for each environment
    
    **Don't:**
    - Hard-code keys in source code
    - Store keys in version control
    - Share keys via email or chat
    - Use production keys in development
  </Accordion>
  <Accordion icon="shield" title="Access Control">
    **Implement:**
    - Role-based access control (RBAC)
    - Principle of least privilege
    - API key scoping and permissions
    - Regular access reviews and audits
  </Accordion>
</AccordionGroup>

## Error Handling

### 🔄 Retry Logic

Implement intelligent retry mechanisms:

```javascript
class PaymentService {
  async processPayment(paymentData, maxRetries = 3) {
    const idempotencyKey = this.generateIdempotencyKey(paymentData);
    
    for (let attempt = 1; attempt <= maxRetries; attempt++) {
      try {
        const response = await this.makePaymentRequest(paymentData, idempotencyKey);
        return response;
        
      } catch (error) {
        // Don't retry on client errors (4xx)
        if (error.status >= 400 && error.status < 500) {
          throw error;
        }
        
        // Don't retry on final attempt
        if (attempt === maxRetries) {
          throw error;
        }
        
        // Exponential backoff with jitter
        const delay = this.calculateBackoff(attempt);
        await this.sleep(delay);
      }
    }
  }
  
  calculateBackoff(attempt) {
    const baseDelay = 1000; // 1 second
    const maxDelay = 30000;  // 30 seconds
    const jitter = Math.random() * 1000;
    
    return Math.min(maxDelay, baseDelay * Math.pow(2, attempt) + jitter);
  }
}
```

### ⚠️ Error Classification

Handle different error types appropriately:

<AccordionGroup>
  <Accordion icon="user-times" title="Client Errors (4xx)">
    **Don't retry these errors:**
    - 400 Bad Request - Fix request format
    - 401 Unauthorized - Check API credentials
    - 403 Forbidden - Verify permissions
    - 404 Not Found - Check endpoint URL
    - 429 Too Many Requests - Implement rate limiting
  </Accordion>
  <Accordion icon="server" title="Server Errors (5xx)">
    **Safe to retry with backoff:**
    - 500 Internal Server Error
    - 502 Bad Gateway
    - 503 Service Unavailable
    - 504 Gateway Timeout
  </Accordion>
</AccordionGroup>

### 📊 Monitoring and Alerting

```javascript
// Implement comprehensive monitoring
class PaymentMonitor {
  constructor() {
    this.metrics = {
      totalRequests: 0,
      successfulRequests: 0,
      failedRequests: 0,
      averageResponseTime: 0
    };
  }
  
  recordRequest(success, responseTime) {
    this.metrics.totalRequests++;
    
    if (success) {
      this.metrics.successfulRequests++;
    } else {
      this.metrics.failedRequests++;
      this.alertOnFailure();
    }
    
    this.updateAverageResponseTime(responseTime);
  }
  
  alertOnFailure() {
    const failureRate = this.metrics.failedRequests / this.metrics.totalRequests;
    
    if (failureRate > 0.05) { // Alert if failure rate > 5%
      this.sendAlert('High payment failure rate detected');
    }
  }
}
```
